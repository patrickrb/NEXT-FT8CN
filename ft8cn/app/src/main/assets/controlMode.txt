Control mode refers to how FT8CN controls the radio during signal transmission.
  VOX: Voice-activated mode triggers transmission.
  CAT: Controls the radio via serial port commands. Different radio models may use different command sets. Most ICOM radios use the CI-V command set, and compatibility between most ICOM models is relatively high. YAESU radio serial commands are generally divided into 3-4 types of command sets, with radios in the same category being compatible with each other.
  RTS: Controls the radio's PTT by toggling the RTS circuit on the serial port. Some older radios (such as the IC-706) cannot control PTT transmission via serial commands and require toggling RTS on/off to control PTT. Selecting this option does not mean serial commands are unused - apart from PTT control using RTS, all other radio control functions still use serial commands. When selecting RTS, you still need to set the radio's address and baud rate to allow the app to control the radio.
  DTR: Similar to RTS, controls the radio's PTT by toggling the DTR circuit on the serial port.
  Generally speaking, even if command sets differ between radio models, most commands within the same manufacturer or series are the same. FT8CN aims for maximum compatibility by using as few serial commands as possible.
  The main serial commands used are: read and set frequency, read and set mode (USB upper sideband), PTT key on, PTT key off.
  My phone doesn't have a serial port? Many modern radios map a serial port through their USB port. If your radio supports this feature, simply connect your phone to the radio using a USB OTG adapter cable.
  Using the IC-705 as an example, the 705's USB maps two serial ports and also creates a virtual sound card. When the phone is connected to the IC-705 via USB cable, the phone can control the radio through the mapped serial ports while simultaneously using the virtual sound card for audio capture and transmission.

Notes for CAT mode:
  1. USB ports distinguish between host and device roles. The radio's USB port operates in device mode, and the phone defaults to device mode as well. To control the radio from the phone, the phone must be switched to host mode, typically achieved through an OTG adapter.
  2. When the radio is transmitting, electrical current may interfere with the USB cable, causing communication interruption between the phone and radio and loss of radio control. A common solution is to add ferrite cores to the USB cable. The best solution is to use a USB isolator, which costs around $5-10.
  3. Using certain OTG adapters or USB hubs may increase the radio's noise floor. Please choose an appropriate adapter.
